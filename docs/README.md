# Arkpad Documentation Hub

This folder contains the complete planning and delivery documentation for Arkpad.

## Contents

- [PRD](PRD.md): Product requirements, goals, scope, and acceptance criteria.
- [Arkpad API](ARKPAD_API.md): The public editor API and extension model.
- [Roadmap](ROADMAP.md): Phased delivery plan.
- [Architecture](ARCHITECTURE.md): Technical design and system boundaries.
- [Setup](SETUP.md): Local development setup and environment checklist.
- [TODO](TODO.md): Actionable task list for implementation.
- [QA Plan](QA_PLAN.md): Testing strategy and release quality gates.

## How to use this folder

1. Start with PRD and align on goals.
2. Confirm architecture choices.
3. Execute tasks from TODO in priority order.
4. Validate against QA Plan before release.

## Ownership

- Product: Define scope and priorities.
- Engineering: Design and implementation.
- QA: Validation and sign-off.
